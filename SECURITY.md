# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in CodeScope, please report it responsibly.

**Preferred method:** Use [GitHub Security Advisories](https://github.com/AlrikOlson/codescope/security/advisories/new) to report the issue privately.

**Alternative:** Open a [GitHub issue](https://github.com/AlrikOlson/codescope/issues) if the vulnerability is not sensitive (e.g., a dependency with a known CVE).

## Scope

This policy covers:

- The `codescope-server` binary (MCP and HTTP modes)
- The web UI frontend
- The setup and installation scripts

## Response Timeline

- **Acknowledgment:** Within 7 days of report
- **Fix timeline:** Best effort, depending on severity

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Impact assessment (what an attacker could do)
- Suggested fix, if you have one
